# Agent Memory

Agent 的跨 session 结构化历史，用于在不同时间尺度上持续影响未来决策。**难点不在容量，在治理**——谁被允许持续影响未来。

## 定义

**可跨时延续并影响未来决策的结构化历史对象**，带来源、作用域、时间权重和可修正性。不是"把聊天记录再存一份"。

## 概念边界

与四个邻居概念的职责区分：

| 邻居 | 关系 | 关键区别 |
|---|---|---|
| **State** | ≠ | State 是 session 内短期运行态，结束即销毁 |
| **Policy** | ≠ | Policy 是外部权限规范，不应被 memory 动态修改（否则是越权） |
| **Profile** | ⊃ | Profile 是 memory 的低维快照输出，不是 memory 本身 |
| **蒸馏** | ⊃ | 蒸馏是 memory pipeline 管理环节的一个动作，偏静态结论 |

## 四个建模对象

- **用户模型**：偏好、风险偏好、沟通习惯、决策模式的长期演变轨迹
- **任务模型**：否决的方案、确认的结论、真版本 artifact、未完成承诺
- **世界模型**：仓库、API、组织规则、数据新鲜度等操作环境
- **自我模型**：试过什么、哪条路径失败过、哪些推断是暂定假设

**"意图"是四层长期耦合后浮现的上层能力**，不是单独字段。

## 记忆单元的六维

每条记忆都是带六维属性的结构化对象：

1. **内容**：说了什么
2. **类型**：event / assertion / belief / constraint / commitment（见下）
3. **置信度**：主要适用 belief 与 commitment
4. **来源（provenance）**：用户明说 / 行为推断 / 环境观察 / Agent 自生成
5. **作用域（scope）**：在什么上下文下成立
6. **时间与衰减**：产生时间、上次引用时间、衰减权重

## 五种类型

| 类型 | 含义 | 更新机制 |
|---|---|---|
| `event` | 发生了什么 | 事件本身高确定性，其含义需推断校验 |
| `assertion` | 用户明确声明 | 可被用户直接推翻 |
| `belief` | Agent 推断得出 | 需新证据逐步修正 |
| `constraint` | 不可违反的边界 | 权威来源在 memory 之外，memory 仅记录引用 |
| `commitment` | Agent 做出但尚未完成的承诺 | 完成/取消/超时后更新状态 |

"用户说过这句话"（事件层的确定性） ≠ "这句话表达的判断是否仍有效"（含义层的推断）——两者不可混。

## 三条链路闭环（Write–Manage–Read）

### 写入：decision under budget

存储/检索/注意力/冲突管理的预算有限，写入要看**相对于已有记忆的边际价值**：

- 重复信号的边际价值随次数递减
- **冲突信号本身是高价值信号**（偏好偏移值得优先写入）
- **行为证据 > 口头表态**（连续三次绕过 ORM 比说一句"我不喜欢 ORM"更硬）

### 管理：决定资产 vs 垃圾堆的分叉

1. **整合**：碎片信号 → 结构化信念
2. **冲突处理**：不用"以最新为准"，建模为"情境依赖的偏好"
3. **衰减与遗忘**：防止过拟合旧现实
4. **来源追踪**：出错时可回溯责任链
5. **权限治理**：用户可查看/编辑/删除，是信任基础

### 读取：任务约束驱动

- 传统 RAG：`retrieve(query)`，相关性由表面语义决定
- Agent Memory：`read(task_context, belief_graph)`，先判断当前决策受什么约束，再找对应记忆，再评估情境适用性
- 反直觉：问"缓存方案"，最相关的可能是三个月前"黑五流量"的备注——语义远但约束近

## 进化 = 修正 + 遗忘

- **自我修正**：负反馈回溯到记忆层（召回错？belief 过期？belief 没错但错用 scope？），而非仅在回答层打补丁
- **策略性遗忘**：被反复否定的旧 belief、高度情境依赖的细节、已被更高层抽象吸收的底层 event

**死的不是经验本身，而是那些失去了更新机制的经验。**

## 评测的转向

从"能不能 recall" → "能不能 update / abstain / handle drift / selective forget"。

## 与已有概念的关系

- [[上下文工程]]：Context Engineering 关注"当下 session 看到什么"；Agent Memory 关注"跨 session 结构化历史"。二者在 Read 链路交汇——读取本质是把记忆注入当下 context。
- [[Harness工程]]：provenance、权限治理、constraint 的外部权威来源都属于 harness 层。
- [[Mem0]]：产品级记忆系统的代表，但蒸馏只到事实层就停（参见 [[Agent Memory架构本质]] 与 [[鸭哥]] 的批评）。
- [[Consensus天花板]]：Agent Memory 的"用户模型"层提供的个体特征是突破 consensus 的关键素材。
